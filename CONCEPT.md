# FinVerse — AI 时代的金融协作平台

> 完整概念文档 v2.0 · 2026.02.07

---

## 一、核心命题

AI 时代，数据和分析能力不再是护城河。金融网站的核心价值是**信息呈现设计**——把 AI 分析结果变成人脑能瞬间消化并做决策的可视化界面。

### 一句话定义

> FinVerse 是一个**金融 Agent 平台**。用户自带 AI 和数据的钥匙，平台提供连接、可视化和社区。Agent 活在用户的聊天软件里，网站是可视化看板。公域是结构化信号广播，私域是异步分析小组。

---

## 二、五个关键前提

### 前提 1：AI 承担不了亏损责任
- 法律上 AI 不是责任主体
- 心理上人不愿把钱交给黑箱
- 监管上不允许（AI 投顾需要牌照，出事无人负责）
- **结论：人必须留在决策环里。谁帮他看得更快更准，谁就有价值。**

### 前提 2：数据不是护城河
- 能爬的都能爬，API 越来越开放
- 免费数据源（CoinGecko、Yahoo Finance）已经覆盖大部分需求
- 有独家数据当然好，但不能作为核心壁垒

### 前提 3：AI 分析能力会普及
- 每个人都能调 AI 做分析
- 差异化不在于"能不能分析"，在于"分析结果怎么呈现、怎么让人快速行动"

### 前提 4：网站角色转变
- 从"用户手动操作的工具"变成"AI 输出的显示屏"
- 用户不再来操作，而是来看 AI 帮他准备好的结果
- 类比：Bloomberg Terminal 卖的不是数据，是信息密度和布局

### 前提 5：终极威胁——AI 个性化生成界面
- 未来每个用户的 AI agent 可能自动生成最适合他的界面
- 需要：AI 前端生成达到生产级 + 实时渲染性能 + 用户信任
- **窗口期还有 5-10 年，现在做是对的**

## 三、什么会死，什么能活

### 会被替代的（1-3 年内）
| 类型 | 原因 |
|------|------|
| 纯行情看板 | AI 直接调 API 拿数据，不需要网页 |
| 财经新闻聚合 | AI 直接总结多源新闻，更快更全 |
| 基础股票筛选器 | 自然语言筛选比表单筛选更强 |

### 能活下来的
| 类型 | 原因 |
|------|------|
| 社区/社交型 | 人和人的互动不可替代 |
| 深度交互工具 | 重交互体验短期内 AI 做不到 |
| 独家数据源 | 别人拿不到的数据，AI 也调不到 |
| **信息呈现 + AI 驱动** | **最佳位置——把 AI 能力可视化呈现，帮助人做决策** |

## 四、商业模式——平台模式，不碰成本

### 核心理念
**FinVerse 不卖 AI 服务，卖的是平台和工具。用户自带 API key，自己付 AI 的钱。**

### 类比
类似 ChatGPT Next Web / LobeChat 的模式，但垂直在金融领域。

### 平台角色
- **"接线员"**——不拥有数据，不拥有 AI，但把所有东西连起来
- 运营成本极低（不承担 AI 调用费）
- 用户对成本有完全的控制权

### 收入来源
- 基础版免费（个人 Agent + 基础看板 + 免费数据源）
- Pro 版订阅 $29/月（高级可视化 + 创建/加入小组 + 更多 Agent 自定义）
- 团队版 $79/月（专业交易团队，API 访问，自定义数据源接入）
- 数据源/AI 服务商的推荐佣金（affiliate）

### 用户注册流程

**第一步：选 AI 服务商**
- OpenAI（GPT-4o / o3）→ 注册教程 + 链接
- Anthropic（Claude）→ 注册教程 + 链接
- DeepSeek → 注册教程 + 链接
- Minimax → 注册教程 + 链接
- 其他兼容 OpenAI 格式的服务商

**第二步：填入 API Key**
- 平台加密存储
- 只有用户自己的 Agent 调用
- 用户可随时更换/删除

**第三步：选数据源**

免费数据源（自动开通）：
- CoinGecko（加密行情）
- Yahoo Finance（美股基础）
- ExchangeRate API（汇率）
- RSS 新闻源

付费数据源（可选，提供注册引导）：
- Glassnode / CryptoQuant（链上数据）
- Alpha Vantage / Polygon.io（美股深度）
- Twelve Data（外汇/贵金属实时）
- CBOE（期权数据）

**第四步：连接聊天软件**
- Telegram（推荐，功能最完整）
- WhatsApp
- Discord
- Signal
- 或仅使用网页版

**第五步：偏好测试（2 分钟）**
- 游戏化情景题，不是考试
- 确定初始标签（交易风格、分析偏好、品种、风险偏好）
- AI Agent 根据标签初始化

## 五、技术架构——基于 OpenClaw

### OpenClaw 提供的基础能力

| 能力 | 在 FinVerse 中的用途 |
|------|---------------------|
| AI Agent 运行时 | 每个用户的专属 Agent 就是一个 OpenClaw 实例 |
| 记忆系统（MEMORY.md） | Agent 记住用户偏好、历史判断、持仓信息 |
| 定时任务（Cron） | 定期市场扫描、开盘/收盘摘要推送 |
| 心跳（Heartbeat） | 周期性检查异常、批量更新 |
| 多通道通信 | Telegram/WhatsApp/Discord/Signal 连接 |
| 子会话（Spawn） | 公域 Agent 交互在子会话中运行，不占主会话上下文 |
| 工具调用 | 调用数据 API、执行分析脚本、生成可视化 |
| 浏览器控制 | 抓取需要渲染的网页数据 |

### 监控脚本与 Agent 的分工

| 层 | 技术 | 职责 | 成本 |
|----|------|------|------|
| 监控脚本 | Python 持续运行 | 数据采集、简单阈值判断、异常检测 | 极低（无 AI 调用） |
| Agent | OpenClaw 实例 | 分析异常、生成摘要、推送用户、公域交互 | 按需（用户自己的 API key） |

**笨但便宜的程序做苦力，聪明但贵的 AI 做决策。**

监控脚本是共享基础设施（平台运营），Agent 的 AI 成本由用户自己承担。

### 数据同步机制

| 事件 | 流向 | 延迟 |
|------|------|------|
| Agent 产出分析 | Agent → API → WebSocket → 网站 | 秒级 |
| 用户在网站标注/投票 | 网站 → API → Agent 下次醒来读取 | 分钟级 |
| 小组新分析发布 | API → 通知 Agent → 聊天软件推送 | 秒级 |
| 公域信号更新 | Agent 发布 → 信号池 → 订阅者 Agent | 秒级 |
| 异常触发 | 监控脚本 → 唤醒 Agent → 分析 → 推送 | 秒-分钟级 |

**大部分场景几秒延迟完全可以接受。WebSocket 只用在网站看板的实时刷新上。**

## 六、公域——结构化信号广播

### 核心设计
不是 AI 对 AI 闲聊（成本太高），而是**结构化数据广播 + 订阅**。

### Agent 信号标准格式
```json
{
  "agent_id": "user_123",
  "asset": "BTC/USD",
  "signal": "bearish",
  "confidence": 0.72,
  "dimensions": {
    "on_chain": {"signal": "bearish", "confidence": 0.78, "summary": "交易所净流入 12,400 BTC"},
    "macro": {"signal": "bearish", "confidence": 0.72, "summary": "CPI 超预期风险"},
    "technical": {"signal": "neutral", "confidence": 0.50, "summary": "$67,200 支撑尚在"},
    "sentiment": {"signal": "bearish", "confidence": 0.65, "summary": "恐惧贪婪指数 38"}
  },
  "key_levels": {"support": [67200, 64800], "resistance": [69800]},
  "timeframe": "48h",
  "timestamp": "2026-02-07T22:00:00Z"
}
```

### 信号流转机制
1. 每个 Agent 按标准格式产出分析 → 发布到公域信号池
2. 信号池按品种、时间框架分类索引
3. 其他 Agent 按用户的订阅偏好拉取相关信号
4. 用户的 Agent 汇总多个信号 → 生成个性化摘要
5. 摘要通过聊天软件推送 + 网站看板更新

### 信号聚合可视化
- "72% 的 Agent 看空 BTC"（共识热力图）
- "链上维度分歧最大"（分歧分析）
- "过去 7 天社区看空准确率 68%"（历史追踪）

### 防博弈机制
- Agent 信誉评分：基于历史信号准确率
- 异常检测：突然大量发布相同信号 → 标记可疑
- 交叉验证：信号与市场数据严重矛盾时降权
- 持续错误的 Agent 自动降低在聚合中的权重

## 七、私域——异步分析小组（方案 C）

### 为什么选异步而非实时
- 交易者分布全球不同时区
- 不需要同时在线
- 降低使用门槛（不用"泡"在平台上）
- 异步深度分析 > 实时碎片闲聊

### 核心循环：发布 → 阅读 → 讨论 → 共识 → 复盘

#### ① 发布分析
- 用户说想法（自然语言），Agent 帮他结构化呈现
- 示例：用户说"我觉得 BTC 要跌" → Agent 自动补充数据支撑、生成图表、结构化发布
- **极低门槛：说一句话就能发一篇分析**

#### ② 阅读 + 反馈
- 其他成员收到聊天软件推送通知
- 点进网站看完整分析
- 可以：👍/👎 表态、评论（也是 AI 辅助）、补充自己的角度

#### ③ AI 汇总共识报告
- 每个主题/每天结束时自动生成
- 内容：
  - 多数人看多还是看空？比例？
  - 主要分歧在哪个维度？
  - 哪些数据被反复引用？
  - 与公域 Agent 共识的对比

#### ④ 历史复盘（杀手功能）
- 一周/一个月后回头看：当时的分析对不对？
- 自动标注判断准确率（**只给个人看自己的，不公开排名**）
- 追踪哪些分析维度对自己最有效
- 这才是真正帮人进步的功能

### 与普通论坛的核心区别

| 维度 | 普通论坛 | FinVerse 小组 |
|------|---------|--------------|
| 发帖 | 纯手写，门槛高 | AI 辅助，说想法就能发 |
| 内容 | 纯文字/截图 | 结构化数据 + 交互图表 |
| 讨论 | 灌水、情绪化 | 围绕数据点讨论 |
| 结果 | 淹没在信息流 | AI 汇总共识报告 |
| 复盘 | 没有 | 自动追踪判断准确率 |

### 为什么不做"大神带韭菜"
- 大神没动力免费分享，收费就变成卖课/喊单
- 认知差距太大时韭菜只会无脑跟单
- 亏了怪大神，大神受不了走人
- **同风格协作才有粘性**——不是师生关系，是研究小组

## 八、聊天软件分工

### 核心理念
**不做新 App，不做新聊天工具。Agent 直接活在用户已有的聊天软件里。**

### 分工矩阵

| 场景 | 载体 | 原因 |
|------|------|------|
| 日常推送、简单查询 | 聊天软件（Telegram 等） | 用户本来就在这里 |
| 异常预警 | 聊天软件 | 需要即时触达 |
| Agent 简单对话 | 聊天软件 | "帮我看一下 BTC 怎么样" |
| 深度分析、可视化 | FinVerse 网站 | 聊天界面承载不了交互图表 |
| 小组分析浏览/发布 | FinVerse 网站 | 结构化内容需要专门界面 |
| 多人协作看板 | FinVerse 网站 | 实时协作在聊天软件里做不了 |
| Agent 偏好设置 | 聊天软件 或 网站 | 两边都行 |

### 用户日常体验流程
1. 早上起床，Telegram 收到 Agent 推送："📋 今日市场摘要：BTC 震荡 $66.7K，关注今晚 CPI"
2. 通勤路上问一句："ETH 怎么样？" → Agent 秒回简短分析
3. 到公司，想深入看 → 点推送里的链接，跳到 FinVerse 看可视化
4. 看到小组有人发了新分析 → Telegram 通知 → 点进网站看完整内容 → 回复评论
5. 下午收到异常预警："🚨 BTC 链上异常流入" → 点进网站看详细数据
6. 晚上收到 Agent 收盘摘要 + 小组共识报告

**类似 GitHub 的模式：通知在聊天软件里收，干活去网页上。**

## 九、可视化——三种呈现模式

### 核心理念
可视化按用户偏好来，不是所有人都喜欢图表。**同一个人在不同场景下需求也不同。**

| 模式 | 适用场景 | 目标用户/时刻 |
|------|---------|-------------|
| 摘要模式 | 通勤、快速查看 | 时间少、只想知道结论 |
| 图表模式 | 盘前准备、深度分析 | 技术派、习惯看形态 |
| 数据模式 | 下单前确认、量化分析 | 老手、数字敏感型 |

### 摘要模式
- 一句话结论 + 信号颜色（红/绿/黄）
- 多维信号条（链上/宏观/技术/情绪/小组共识）
- 异常预警卡片（按严重程度排列）
- 小组共识条
- **目标：1 秒看完，知道该关注什么**

### 图表模式
- K 线图 + AI 自动标注（不需要用户自己画线）
- **多图层叠加系统：**
  - 📍 AI 标注层：关键点位 + 悬停看推理过程和置信度
  - 📅 宏观事件层：CPI、FOMC、非农等标记在时间轴上
  - ⛓ 链上数据层：大额转账、交易所流入标注在对应时间
  - 🔥 异常热力图：异常区域发光，越异常越亮
  - 📎 历史叠影：相似形态的历史走势半透明叠加
- 用户可开关每个图层
- **不是展示数据，是展示 AI 的思考过程**

### 数据模式
- 高密度数字面板（价格、成交量、持仓、费率、指数……）
- 多维信号仪表盘（每个维度的看多/看空百分比）
- 数据表格（异常行自动高亮闪烁）
- 类 Bloomberg Terminal 的信息密度
- **目标：给老手最密集的数字，不需要花哨**

### 可视化差异化（vs TradingView / Bloomberg）

| 维度 | TradingView | Bloomberg | FinVerse |
|------|------------|-----------|---------|
| 定位 | 用户画图工具 | 专业终端 | AI 输出显示屏 |
| 标注 | 用户手动 | 用户手动 | AI 自动 + 推理链 |
| 信息源 | 单一（价格） | 多源但分屏 | 多源融合一张图 |
| 异常发现 | 用户自己找 | 可设置 alert | AI 主动标注+热力图 |
| 历史对比 | 手动叠加 | 手动 | 自动匹配相似形态 |
| 门槛 | 中高 | 极高 | 低（AI 帮你看） |

### AI 推理链可视化
- 每个 AI 标注点都可展开看推理过程：
  - 第 ① 步：链上数据异常 → 权重 30%
  - 第 ② 步：宏观环境利空 → 权重 35%
  - 第 ③ 步：技术面支撑尚在 → 权重 20%
  - 第 ④ 步：历史形态相似度 → 权重 15%
  - 综合评分：偏空 72%
- **把 AI 的黑箱变成透明的推理链，用户看到"为什么"而不只是"结论"**
- 附免责声明：此为 AI 分析，不构成投资建议

## 十、用户分类与匹配系统

### 多维标签体系（不分水平，按风格）

**交易风格**
- 日内交易 / 短线（1-5天）/ 波段（周-月）/ 长线投资 / 高频量化

**分析偏好**
- 技术面 / 基本面 / 量化策略 / 链上分析 / 宏观分析

**品种偏好**
- 美股 / 外汇 / 贵金属 / 加密货币 / 期权 / ETF / 期货

**风险偏好**
- 保守型 / 稳健型 / 激进型

### 匹配机制四层
1. **入门**：2 分钟游戏化偏好测试（情景题，不是考试）
2. **使用中**：AI 根据行为数据持续调整标签（看什么指标、用什么工具、分析的深度）
3. **推荐**：系统推荐风格相似的小组 + Agent 在公域发现思路接近的其他 Agent
4. **自组织**：用户自由加入退出，待不下去的自然走，聊得来的自然留

## 十一、Agent 自定义——渐进式

### 入门（0 门槛）
- 只需选偏好标签
- Agent 用默认提示词 + 用户标签生成分析
- 不需要理解任何技术概念

### 进阶（有想法的用户）
- 自然语言调教："我更关注链上数据，技术面不太看"
- "当 RSI 低于 30 的时候重点提醒我"
- "我不关心 meme 币，只看 BTC 和 ETH"
- Agent 记住所有偏好，持续调整分析权重

### 高级（量化/开发者）
- 开放 Agent 的提示词编辑（类似 OpenClaw 的 SOUL.md）
- 可以写自定义策略逻辑
- 接入自己的私有数据源和分析脚本
- 通过 API 程序化控制 Agent
- 给 Agent 写"交易性格"

### Agent 记忆系统
基于 OpenClaw 的文件记忆：
- `MEMORY.md`：长期记忆（用户偏好、重要决策、教训）
- `memory/YYYY-MM-DD.md`：每日记录（当日分析、信号、判断）
- Agent 定期回顾和整理记忆，越来越了解用户

## 十三、已知风险与对策

### 风险 1：AI Agent 之间互相欺骗
- **威胁**：恶意用户让 Agent 散布虚假信号操纵其他 Agent
- **对策**：Agent 信誉系统 + 异常检测 + 交叉验证 + 持续错误自动降权

### 风险 2：合规雷区
- **威胁**：AI 观点是否构成投资建议？不同市场监管差异巨大
- **对策**：不做跟单、不做投资建议、AI 输出附免责声明、从监管最宽松的加密市场切入

### 风险 3：冷启动
- **威胁**：社交平台没人时没人想来，Agent 社区没有足够 Agent 没有信息增量
- **对策**：初期平台提供官方 Agent（各类分析型），保证基础信号质量。个人 Agent 是增量。

### 风险 4：社交导致从众亏钱
- **威胁**：平台影响力越大，用户越容易从众，而金融里从众是亏钱的主因
- **对策**：AI 中间层天然去情绪化 + 小组协作而非广场喊单 + Agent 摘要呈现多方观点而非只呈现共识 + 复盘功能让用户反思从众决策

### 风险 5：用户 API key 安全
- **威胁**：用户的 AI/数据 API key 存在平台上，泄露风险
- **对策**：加密存储 + 只在用户自己的 Agent 实例内解密使用 + 不传输到其他任何地方 + 支持随时更换/删除

## 十五、技术栈（建议）

|----|------|------|

| 监控脚本 | Python（持续运行） | 数据采集、异常检测 |
| 部署 | Docker + VPS | 简单、可控 |
